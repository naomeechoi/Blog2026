### 파일에 텍스트 쓸 때 \0 널 문자 넣지 않는다.
파일에 텍스트를 쓸 때 \0을 넣으면 바이트 변환뒤 65 00 00 00 00 .. 이런식으로 쓰이게 된다.
텍스트 편집기는 이를 UTF-16으로 읽어 ANSI나 UTF-8로 텍스트 파일 작성을 의도했을 때 파일이 깨지게 된다.
따라서 널을 빼고 텍스트 글자 갯수를 맞춰서 적는 게 좋다.

```
	char outputData[256] = {};
	int len = sprintf_s(outputData, 256, "id=%d score=%d attack=%f", id, score, attack);

	size_t writtenSize = fwrite(outputData, sizeof(char), 256, file); -> 문제
	size_t writtenSize = fwrite(outputData, sizeof(char), len, file); -> 해결 256대신 len을 넣어줌
```

### for (const int& const item : array) 에서 뒤에 붙는 const가 의미가 없는 이유
& 참조 타입 자체가 다른 값을 다시 가리킬 수 없기 때문에 뒤에 붙는 const는 의미 없다.
예를 들어
int a = 10;
int b = 20;
int& c = a;
c = b; 를 하면 c는 b를 가르키는게 아닌 a의 값을 변경한다.
즉 다른 값 가리킬 수 없다.
