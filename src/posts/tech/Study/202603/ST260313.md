3) 감마보정이란?
컴퓨터가 계산하는 밝기와 사람 눈이 느끼는 밝기가 다르다. 감마 보정이란 화면 밝기를 사람 눈에 자연스럽게 보이도록 조정하는 과정이다.
감마보정 때문에 색상 데이터의 원래 값이 아닌 보정된 다른 값이 된다. 따라서 텍스처가 데이터로 쓰이는 경우라면 감마 보정을 빼기 위해 Mask 모드로 사용해야한다.

4) 아핀 변환
선형 변환은 원점을 고정한 채 크기/회전/비율을 조절하는 변환을 말한다. 아핀 변환은 여기에 평행 이동 (원점 이동)을 가능하게 한 변환을 말한다.
3D 그래픽스에서 크기 변경, 회전, 이동, 비틀기 등을 표현할 때 쓴다.

6) 동적 배열을 미리 vector.reserve 하는 이유
std::vector는 내부 배열이 꽉 차면 재할당 및 복사(이동)하는 작업이 발생한다. 이는 성능 저하로 이어지므로 미리 메모리를 사용할 만큼의 사이즈로 할당해두면 재할당을 줄일 수 있다.
메모리 할당 횟수, 불필요한 데이터 복사 방지, 성능 향상을 위해 하는 작업이다.
</br></br>
7) new/delete와 malloc/free의 차이점
new는 C++ 전용 연산자로 생성자를 호출하며 타입에 맞는 포인터를 반환해준다.
즉 내부에서 메모리 할당과 생성자 호출을 동시에 해주고 타입에 맞는 포인터까지 반환해주는 것이다.
또 실패시 예외를 발생시킨다.
반면 malloc은 C 표준 라이브러리 함수로, 생성자 호출을 해주지 않고 내부에서 메모리 할당만 수행한다.
void*를 반환하기 때문에 캐스팅이 필요하고, 메모리 할당 실패시 NULL이 반환된다.
메모리 풀처럼 우선은 메모리 공간만 필요하고 메모리 공간이 사용되는 시점에 타입이 결정된다면 malloc을 사용하는 게 좋다.
</br></br>
8) SIMD (Single Instruction Multiple Data)
CPU 아키텍처에서 지원하는 병렬 처리 방식 중 하나이다. 한 개의 명령어로 여러 개의 데이터를 동시에 처리할 수 있다.
예를 들어, 4개의 32비트 숫자 덧셈을 한 명령어로 처리하는 것으로 처리 속도를 대폭 향상시킬 수 있다.
그래픽 처리, 물리 시뮬레이션, 영상 처리 등 벡터화가 가능한 반복 계산에 최적화된다.
MMX, SSE는 SIMD 명령어 집합으로 MMX는 정수 연산 위주로 가능했고 SSE는 부동소수점 연산을 지원한다.
CPU는 행기준으로 데이터를 저장하고 GPU는 열기준으로 데이터를 저장한다. 데이터가 연속적으로 저장된다는 것은 SIMD 레지스터에 데이터를 한 번에 넣을 수 있다는 뜻이다.
SIMD는 메모리에서 데이터를 한 번에 읽어와 병렬 연산을 하는 것이기 때문에 메모리 레이아웃이 맞지 않으면 캐시 미스나 불필요한 데이터 이동이 생긴다.
따라서 CPU와 GPU의 SIMD는 설계가 다르게 되어 있다. SIMD는 하드웨어 아키텍처와 메모리 접근 패턴에 맞춰 만들어져 있다.

9) 왜 CMake는 윈도우와 맥 모두에서 C++ 코드가 컴파일 가능하게 하나?
CMake는 각 플랫폼(Windows, macOS, Linux)의 네이티브 빌드 도구(MSBuild, Makefile 등)를 생성한다.
CMake 자체가 컴파일러가 아니고, 컴파일러 호출 명령어를 생성하는 역할을 한다.
덕분에 다양한 OS, 컴파일러 환경에서 일관된 빌드 구성을 유지할 수 있다.
</br></br>

12) FileMD5
파일의 MD5 해시값을 의미한다. 파일의 무결성을 확인할 때 사용한다.
파일 데이터를 입력하면 고정 길이 해시값을 생성한다. 파일 내용이 1비트라도 바뀌면 MD5 값이 완전히 바뀐다.
파일 내용이 같은지 빠르게 비교한다.
공격자가 의도적으로 해시 충돌을 일으킬 가능성이 있어 보안 용도로 더 이상 안전하지 않다.
